# M@S Studio Design System for Claude Design

This directory is the design system package for M@S Studio — Adobe's platform for authoring
commerce content: product cards, pricing fragments, promotional offers, and subscription plans.

## What's in here

| File | Purpose |
|---|---|
| `components.md` | Every SWC component available in Studio, with real usage examples |
| `tokens.md` | Spectrum 2 token reference + M@S-specific `--mas-*` overrides |
| `patterns.md` | Non-negotiable Lit/SWC conventions Claude Design must follow |
| `scaffold.html` | Minimal working Studio shell — base for every prototype |
| `examples/` | 5 real Studio UI patterns as standalone browser-openable HTML files |
| `scripts/sync.js` | Keeps `components.md` in sync with `swc.js` on every build |

## Setting up Claude Design (one-time per person)

1. Open [Claude Design](https://claude.ai/design)
2. Create a new project
3. Connect this GitHub repository
4. Scope the context to this directory: `studio/design-system/`
5. Save the project as **"M@S Studio Design System"**

## Generating a Studio feature mockup

Paste this starter prompt, then describe your feature:

```
You are designing UI for M@S Studio — Adobe's platform for authoring
commerce content: product cards, pricing fragments, promotional offers,
and subscription plans that appear across adobe.com and Creative Cloud.

Use scaffold.html as your base. All components must come from components.md.
Follow every rule in patterns.md. Reference examples/ for real Studio patterns.

Every UI you generate is used by content authors and commerce managers,
not end consumers — prioritize clarity, density, and editorial efficiency
over marketing aesthetics.
```

### Example prompts

- *"Design a bulk-tag dialog. Authors select multiple fragments and apply taxonomy tags in bulk. Show a tag picker, affected fragment count, and confirm/cancel actions."*
- *"Design a locale picker panel that shows available locales as a filterable list with publish status indicators."*
- *"Design an empty state for the fragment table when no results match the current search query."*

## What Claude Design produces

Each session outputs:
- **HTML prototype** — renders in browser using real SWC components and Spectrum 2 tokens
- **Annotated spec** — which components, slots, CSS custom properties, and Lit patterns to use
- **Lit scaffold** — starter `.js` file following `mas-` conventions from `patterns.md`

Use Claude Design's **"Hand off to Claude Code"** button to open a Studio session with the prototype + spec loaded as context.

## Keeping this up to date

Only two files need maintenance:

- **`components.md`** — update in the same PR whenever you add or remove an import in `studio/src/swc.js`
- **`scaffold.html`** — auto-regenerated by `scripts/sync.js` on every build

Everything else is set-and-forget until a major Studio architectural change.
